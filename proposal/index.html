From Mesh to Render

CS184 Spring 2026 Final Project Proposal

Team members: Carolann Dong, Jocelyn Tao, Nicole Jin, Jessica Ng

Link to proposal webpage: https://jessican1212.github.io/cs184finalproject/proposal/index.html

Summary

This project implements a mesh editing and scene rendering pipeline that progressively refines a 3D model step by step. We first take a mesh built in Blender, apply a remeshing algorithm to regularize anisotropic triangles into more uniform ones, run subdivision to produce smoother geometry, and render the final result with texture support in our path tracer. Each stage produces a visible improvement, demonstrating the full journey from raw mesh to polished final product.

Problem Description

Raw meshes from Blender or other sources often have uneven, poorly shaped triangles. This directly affects both the stability of downstream processing operations and overall visual quality of our objects. Our project builds a multi-step pipeline to fix this: we remesh to clean up triangle quality, subdivide to add smooth detail, and then render the result with textures in our path tracer. The main challenges are implementing remeshing that improves triangles without distorting the original shape, choosing a subdivision sceheme that works well on the remeshed output, and integrating UV-based texture lookups into our existing path tracing framework. We extend our HW2 mesh editor and HW3 path tracer as starting points and rely on work from external research papers at each stage of the pipeline. Another challenge is ensuring that our pipeline works end-to-end, which we will address by testing the full multi-step process early on with a simple mesh to ensure file formats work between independent stages.

Goals and Deliverables

What we plan to deliver:

We plan to deliver three tools that together take a mesh from Blender and progressively refine it into a final rendered image. The first stage applies isotropic remeshing to regularize triangle quality. The second applies a subdivision scheme to add smooth geometric detail. The third adds texture support to our path tracer for realistic rendering. We will evaluate our system by doing before and after comparisons at each stage, demonstrating clear improvements. Ideally, we want to display intermediate results to show how each stage contributes to the final rendered scene.

What we hope to deliver:

If we have enough time, we plan to test our model on more complicated and diverse meshes created in Blender. We would also like to add more textures to handle more realistic rendering scenarios and experiment/compare the effects of multiple subdivision and smoothing techniques.

Schedule

  • Week 1: Research. Read the Botsch & Kobbelt paper and Hao Li lecture slides to understand the remeshing algorithm. Research and choose which subdivision scheme to implement (mostly likely either Catmull Clark, butterfly, or root 3). Read the Cornell lecture slides and understand how to add textures to the HW3 path tracer.
  • Week 2: Implementation. Implements the remeshing loop. Implements the subdivision scheme of choice. Get basic texture mapping working in the path tracer. Create 3-4 test meshes in Blender and export files to later work with.
  • Week 3: Integration and debugging. Connect all stages together by feeding files from Blender to remeshing then subdivision then texture path tracing. Fix any integration issues.
  • Week 4: Render final demo images (intemediate stages and final products). Complete the final project writeup, presentation, video demo, and whatever else needs to be delivered.
