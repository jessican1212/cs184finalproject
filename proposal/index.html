MESH EDIT PIPELINE

CS184 Spring 2026 Final Project Proposal

Team members: Carolann Dong, Jocelyn Tao, Nicole Jin, Jessica Ng

Summary

This project implements a mesh editing pipeline that progressively refines a 3D model step by step. We first take a mesh built in Blender, apply a remeshing algorithm to regularize anisotropic triangles into more uniform ones, run subdivision to produce smoother geometry, and render the final result with texture support in our path tracer. Each stage produces a visible improvement, demonstrating the full journey from raw mesh to polished final product.

Problem Description

Raw meshes from Blender or other sources often have uneven, poorly shaped triangles. This directly affects both the stability of downstream processing operations and overall visual quality of our objects. Our project builds a multi-step pipeline to fix this: we remesh to clean up triangle quality, subdivide to add smooth detail, and then render the result with textures in our path tracer. The main challenges are implementing remeshing that improves triangles without distorting the original shape, choosing a subdivision sceheme that works well on the remeshed output, and integrating UV-based texture lookups into our existing path tracing framework. We extend our HW2 mesh editor and HW3 path tracer as starting points and rely on work from external research papers at each stage of the pipeline. Another challenge is ensuring that our pipeline works end-to-end, which we will address by testing the full multi-step process early on with a simple mesh to ensure file formats work between independent stages.

Goals and Deliverables

What we plan to deliver:

We plan to deliver a complete end-to-end mesh editing pipeline that takes a mesh from blender, improves it's quality, and produces a final rendered image using our path tracer and texture support. The pipeline will be seperated into three stages: remeshing / mesh regularization, a subdivision scheme, and texture support for realistic rendering. We will evaluate our system by doing before and after comparisons, demonstrating clear improvements.

What we hope to deliver:

If we have enough time, we plan to test our model on more complicated and diverse meshes created in Blender. We would also like to add more textures to handle more realistic rendering scenarios and experiment on better subdivision and smoothing techniques. Finally, we want to display intermediate results so that we'll be able to show how each part contributes to the final image.
