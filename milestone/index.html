For new subdivision schemes, we've read up on some possible new schemes that were not covered in homework 2. So far, we've implemented the butterfly subdivision scheme by building off of the code provided in homework 2. We mainly accomplished this by replacing the MeshResampler::upsample function with a MeshResampler::buttefly function. We learned that butterfly subdivision differs from the original loop subdivision scheme we implemented in a few ways, and we implemented the butterfly subdivison as follows:

  • First, we keep all original vertices where they are. This is different because loop subdivision ultimately moves vertices. For each existing vertex V with n neighbors, the original scheme computes a new position (1 − nβ)·V + β·(sum of neighbors).
  • Then, for each edge, we compute a midpoint M. For each edge, compute one new point M. We get that edge's endpoints (A and B), the opposite vertices of the two triangles that share that edge (C and D), and four other vertices that form triangles with edges AC, BC, AD, and BD (we call them E, F, G, and H). We then update M as ½(A + B) + ⅛(C + D) − ¹⁄₁₆(E + F + G + H).
  • Then, we simply split every original edge, flipped new edges that connect an old vertex to a new vertex, and update the vertex positions. These steps are similar to the original scheme.
We also added the ability to use this scheme by pressing the 'B' key.
